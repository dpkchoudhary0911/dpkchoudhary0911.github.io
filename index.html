Vector Run — Game Details

Fast microgame, 3 lanes, step-by-step rounds. Designed for short sessions and high volume.

Core mechanics

  • Choose one of three lanes each round.
  • Each step may spawn an obstacle in a lane.
  • Hit an obstacle → bust. Survive → multiplier increases.
  • Cash out anytime; multiplier grows per step.

Tunable parameters

  • Obstacle probability (easy/med/hard)
  • Step multiplier factor (controls house edge)
  • Tick rate (400–800ms) to balance volume vs control

Provably Fair — How it works

Every round is reproducible from cryptographic inputs so players and auditors can verify outcomes.

  1. Server publishes server_seed_hash = SHA256(server_seed) before rounds.
  2. Player provides client_seed (or one is auto-generated).
  3. Compute HMAC_SHA256(server_seed, client_seed + ':' + nonce) to produce deterministic bytes.
  4. Map bytes into obstacle spawn & lane selection. Reveal server_seed after round for verification.

Example (Node.js):

// Node.js example:
const crypto = require('crypto');
function hmac(serverSeed, clientSeed, nonce){
  return crypto.createHmac('sha256', Buffer.from(serverSeed,'hex'))
               .update(`${clientSeed}:${nonce}`).digest('hex');
